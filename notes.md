\# Git and GitHub Study Notes



\## 1. What is Version Control?

Version control is a system that records changes made to files over time. 

It allows you to see previous versions, track progress, and restore older versions if needed.



\## 2. What is Git?

Git is a distributed version control system. 

It helps developers track changes in their projects and manage different versions of their files.



\## 3. What is GitHub?

GitHub is an online platform that hosts Git repositories. 

It allows developers to store projects in the cloud and collaborate with others.



\## 4. Basic Git Commands



\- `git clone` – Copies a repository from GitHub to your computer.

\- `git add` – Stages changes before committing.

\- `git commit -m "message"` – Saves changes with a clear message.

\- `git push` – Uploads local changes to GitHub.

\- `git pull` – Downloads changes from GitHub.

\- `git checkout -b branch-name` – Creates and switches to a new branch.



\## 5. Why Version Control is Important



\- Keeps track of all changes

\- Prevents losing important work

\- Makes teamwork easier

\- Allows safe experimentation using branches

\- Helps organize project development



\## 6. Branching and Pull Requests



Branches allow developers to work on new features without affecting the main project.

A pull request is used to review changes before merging them into the main branch.

\## 7. What I Learned



Through this project, I learned how to use commits, branches, and pull requests to manage changes safely.

Git helps track progress and makes collaboration more organized.

\## 8. Real-World Workflow



In real projects, developers create branches to work on features separately.

They then open pull requests to review and merge changes safely.
